Unity Cursor-style Chatbot Plugin Plan

1. Objectives
   - Create a Unity Editor plugin that replicates Cursor’s chatbot experience via OpenRouter.
   - Give the model read/write access to scenes, assets, scripts, and project metadata through editor automation services that respect safety boundaries.
   - Surface intuitive UI controls for scope selection, conversation history, and manual confirmation of edits.

2. Key components
   - **Editor UI**: Dockable `EditorWindow` with prompt input, streaming response pane, context toggles, and status indicators.
   - **Context provider**: Services collecting scene graphs, serialized GameObjects, folder listings, and script snippets to include in prompts.
   - **Action executor**: Interpreter mapping model instructions into UnityEditor operations (asset edits, scene adjustments) wrapped in `Undo.RecordObject` and preview modes.
   - **OpenRouter connector**: HTTP client that handles authentication, prompt formatting, streaming responses, and error retries.

3. Workflow
   - User opens the plugin, configures API key, selects scope (scene/folder/project), and submits a prompt.
   - Plugin gathers relevant context, builds a templated system prompt, and dispatches it to OpenRouter while streaming replies.
   - Model responses either produce text guidance or structured edit commands; the executor validates and applies edits after user confirmation.
   - Every mutation is logged, reversible via Undo, and optionally previewed before committing to disk.

4. Implementation milestones
   - Set up Unity package folder structure (`Editor/`, `Runtime/`, `Resources/`) and stub Editor window.
   - Implement UI (chat log, prompt input, context toggles, action buttons) and preference panel for keys and limits.
   - Wire up OpenRouter client with prompt templates, rate limits, and streaming display.
   - Build context-gathering helpers (scene serializer, folder inspector, script reader) with configurable depth.
   - Create instruction parser/executor that safely maps commands (create GameObject, edit script) to Unity API calls.
   - Add telemetry/logging, caching for prompts, and documentation/help overlay.

5. Suggested immediate steps
   1. Choose Unity LTS version and scaffold the Editor package.
   2. Draft UI mockups and define how context is represented in prompts.
   3. Research OpenRouter’s models (e.g., Claude 3 vs GPT) for streaming edits and confirm authentication flow.
